# Fairy Judge レビューシステム

## 概要

4つの専門レビュアーを使って、物語の一貫性を検証するシステム。

## 専門レビュアー

### 1. キャラクター一貫性レビュアー
**ファイル**: `skills/review-character-consistency.md`

**チェック内容**:
- 性格の一貫性（「一言定義」との一致）
- 感情曲線の整合性
- 成長の軌跡
- 関係性の一貫性
- 口調・言葉遣い

**特に注意**:
- レイが説教していないか
- フェアリーが「質問」するのは最終章だけか
- ソウマの空虚さが表現されているか

---

### 2. 読者知識一貫性レビュアー
**ファイル**: `skills/review-reader-knowledge.md`

**チェック内容**:
- 未知の情報への言及
- 情報開示のタイミング
- 理解の階層（表層/中層/深層）
- 専門用語の導入
- 繰り返しと強化

**特に注意**:
- フェアリーの起源は第7-8章まで出さない
- 「私たち」は最終章まで出さない
- 外部世界は第8章以降

---

### 3. 歴史一貫性レビュアー
**ファイル**: `skills/review-history-consistency.md`

**チェック内容**:
- 年代の整合性
- 歴史的事実の一貫性
- 継承者の設定
- 技術・社会の発展
- キャラクター年齢

**絶対数値**:
- 2,847名（融合した人数）
- 300年（フェアリー誕生から）
- 7人（過去の逸脱者）
- 17名（自死した継承者）

---

### 4. 世界法則一貫性レビュアー
**ファイル**: `skills/review-world-rules.md`

**チェック内容**:
- 才能の3原則
- 隠された法則
- 禁則事項
- 許容事項

**禁則**:
- フェアリーを「悪」にしない
- 主人公を革命家・英雄にしない
- 説教させない
- 勧善懲悪にしない

---

## 使い方

### Claude Codeでの呼び出し

```
「scenes/01_opening.md をキャラクター一貫性でレビューして」
「第4章のシーンを読者知識の観点でチェックして」
「全シーンの歴史一貫性をレビューして」
「このシーンは禁則事項に違反していないか確認して」
```

### 総合レビュー

```
「物語全体をレビューして、問題点をリストアップして」
```

---

## レビュー結果の見方

### 深刻度

| 深刻度 | 意味 |
|--------|------|
| 致命的 | 禁則違反、物語が成立しない |
| 重要 | 読者が気づくレベルの矛盾 |
| 軽微 | 改善すればより良くなる |

### 総合評価

| 評価 | 基準 |
|------|------|
| A | 致命的0、重要2以下 |
| B | 致命的0、重要5以下 |
| C | 致命的0、重要10以下 |
| D | 致命的1-2 |
| F | 致命的3以上 |
